# Selective CodeQL value benchmark — results

Answers issue #56 (cheap core). Measures whether the mapper's selective CodeQL
enrichment adds trustworthy value evidence without imposing cost on unrelated runs.
Generated by `scripts/benchmark_codeql_value.py` (opt-in; not in the default unittest
run). Environment: Windows 11, pyenv-win Python 3.10.5, CodeQL CLI 2.26.0,
`codeql/python-all@7.2.1`.

Scope note: this is the cheap, high-signal core — hand-built ground-truth fixtures
covering the guarantees that change whether an agent should trust the evidence. The
expensive tail (pinned real-world repos, precision/recall at scale, peak-resource
profiling, a large sanitizer/wrapper corpus) is deliberately **not** built: CodeQL is
already gated to the rare hard-trigger path, and the engine itself is GitHub's mature,
independently-validated taint library — the skill only adds sink selection plus a
targeted local-flow query. Re-open the full sweep only if CodeQL is promoted to a
default path.

## Fixtures and results

| Fixture | What it is | Ground truth | Result |
|---|---|---|---|
| `tp_dynamic` | `path = root / f'{tenant}.csv'; pd.read_csv(path)` | flow exists | **1 edge** — `VALUE_FLOWS_TO READS_FILE@io.py:4` ✓ |
| `tn_constant` | `pd.read_csv('static_reference.csv')` (literal sink) | no flow | **0 edges** — no fabricated provenance ✓ |
| `xfn_multifile` | source in `a.py` → helper `b.read_it(path)` → `pd.read_csv` in `b.py` | flow crosses a call | **1 edge at the sink** (`READS_FILE@b.py:3`, source = the `path` parameter) |
| `off_by_default` | same code, `--codeql auto --codeql-intent mapping` | no hard trigger | `action=skip`, no build, 0 edges ✓ |
| `degradation_off` | same code, `--codeql off` | disabled | no build, 0 edges, structural graph intact ✓ |

Timings (build-mode=none, per fixture): build ≈ 9.3 s, query ≈ 19–21 s, DB ≈ 3.5 MB.

## Findings

- **True paths found, zero false paths on the trap.** The one metric that matters most —
  false-path rate — is 0 on the constant-sink trap: the mapper does not emit an artifact
  provenance edge for a resolved literal, so an agent is not handed plausible-but-wrong
  evidence.
- **Selectivity holds.** `mapping` intent with no hard trigger runs no CodeQL work
  (`action=skip`), and `--codeql off` degrades to the structural graph with zero cost —
  the off-by-default and graceful-degradation acceptance criteria are met and measured.
- **Cross-function boundary is intra-procedural (by design).** For a multi-file pipeline
  the query flags the artifact read at its **local** source (the `path` parameter inside
  `b.read_it`) but does **not** stitch the provenance chain back across the call into
  `a.py`. This matches the query's identity — `code-mapper/targeted-local-artifact-flow`,
  the *local* artifact-flow query. Consequence for callers: a CodeQL edge reliably points
  at a real artifact sink and its immediate local source; it is **not** a full
  cross-module taint chain. Whole-program taint would need a different (global) query and
  a different cost/latency budget, out of scope for the selective path.

## Acceptance (issue #56)

| Criterion | Verdict |
|---|---|
| Selective CodeQL materially improves useful path evidence on intended tasks | **Met** — true dynamic flows surfaced with sink + local source; 0 false paths on the trap |
| Off by default for unrelated mapper work | **Met** — `mapping`/no-trigger → `skip`, no build (see also `codeql-benchmark-results.md`: default path ≈ 0–1 ms) |
| Absence or failure degrades explicitly to structural output | **Met** — `--codeql off` and build failure both return the structural graph |
| No new standalone security-provider adapter | **Met** — enrichment stays inside the mapper; no new provider |

All five automated checks in `benchmark_codeql_value.py` pass.
