# Design token guidance

Use lightweight tokens when creating no-template visual artifacts. Tokens keep visual choices deliberate and repeatable without forcing a full design system.

## Token Types

- Color: background, surface, text, muted text, border, accent, success, warning, danger.
- Type: body family, heading scale, body size, caption size, line height.
- Space: page margin, section gap, item gap, control padding.
- Radius: small component radius and larger media radius.
- Shadow: avoid unless it clarifies layering.
- Motion: duration and easing for meaningful transitions only.

## Rules

- Define tokens before styling many components.
- Use semantic names rather than color names when possible.
- Keep palettes compact and avoid one-note screens dominated by a single hue family.
- Prefer contrast and hierarchy over decorative gradients.
- Record the token decisions in the artifact or evidence notes when design quality matters.

## Why This Upskills Codex

Tokens make Codex explain its design choices. In no-template work, that prevents random styling drift and helps future revisions preserve the same visual language.
