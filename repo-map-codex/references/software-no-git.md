# Software Without Git Map

Capture:

- root folder and allowed write scope;
- source files, scripts, configs, notebooks, and data files;
- backup or snapshot plan before implementation;
- commands for tests, builds, scripts, notebooks, or smoke checks;
- dependencies and runtime requirements;
- generated outputs and where they should land;
- manual validation needs;
- rollback by restoring snapshots.

When no reliable command validation exists, say so explicitly and propose the smallest useful validation harness.
