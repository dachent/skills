# Mixed Business-Coding Map

Capture both business and technical surfaces:

- business deliverables and stakeholders;
- source documents, data, and approval criteria;
- code, scripts, notebooks, configs, and generated outputs;
- command validation and manual acceptance checks;
- file snapshots or versioning approach;
- privacy and data handling constraints;
- dependencies between code outputs and business artifacts.

The map should make it clear which artifacts are inputs, which are generated, and which are final deliverables.
